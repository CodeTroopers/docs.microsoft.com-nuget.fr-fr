<span data-ttu-id="9054a-106">Un mécanisme à travers lequel les développeurs peuvent créer, partager et consommer des bibliothèques de code utiles est un outil essentiel pour toutes les plateformes de développement modernes.</span><span class="sxs-lookup"><span data-stu-id="9054a-106">An essential tool for any modern development platform is a mechanism through which developers can create, share, and consume useful libraries of code.</span></span> <span data-ttu-id="9054a-107">Ces bibliothèques sont généralement appelées « packages », car elles peuvent contenir du code compilé (sous forme de DLL) ainsi que d’autres types de contenus qui peuvent être nécessaires dans les projets qui utilisent ces bibliothèques.</span><span class="sxs-lookup"><span data-stu-id="9054a-107">Such libraries are typically referred to as "packages" because they can contain compiled code (as DLLs) along with other content that might be needed in the projects that consume those libraries.</span></span>

## <a name="tracking-references-and-restoring-packages"></a><span data-ttu-id="9054a-174">Suivi des références et restauration de packages</span><span class="sxs-lookup"><span data-stu-id="9054a-174">Tracking references and restoring packages</span></span>

<span data-ttu-id="9054a-175">Comme les projets peuvent facilement être déplacés entre les ordinateurs des développeurs, entre les référentiels de contrôle de code source, entre les serveurs de builds, etc., il est très peu pratique de conserver les assemblys binaires des packages NuGet directement liés à un projet.</span><span class="sxs-lookup"><span data-stu-id="9054a-175">Because projects can easily move between developer computers, source control repositories, build servers, and so forth, it's highly impractical to keep binary assemblies from NuGet packages directly bound to a project.</span></span> <span data-ttu-id="9054a-176">Non seulement ceci rendrait chaque copie du projet inutilement surchargée (et gaspillerait donc de l’espace dans les référentiels de contrôle de code source), mais cela rendrait aussi très difficile la mise à jour des fichiers binaires du package vers des versions plus récentes, car la mise à jour devrait être effectuée sur toutes les copies du projet.</span><span class="sxs-lookup"><span data-stu-id="9054a-176">Not only would this make each copy of the project unnecessarily bloated (and thereby waste space in source control repositories), it would also make it very difficult to update package binaries to newer versions as this would have to be done across all copies of the project.</span></span> 

<span data-ttu-id="9054a-177">Au lieu de cela, NuGet gère une liste des références des packages dont un projet dépend (y compris les dépendances des niveaux supérieurs et des niveaux inférieurs), et offre la possibilité de restaurer à la demande tous les packages référencés, comme décrit dans [Restaurer des packages](Consume-Packages/Package-Restore.md).</span><span class="sxs-lookup"><span data-stu-id="9054a-177">Instead, NuGet simply maintains a reference list of the packages upon which a project depends (including both top-level and down-level dependencies), and provides the means to restore all referenced packages upon request as described on [Package restore](Consume-Packages/Package-Restore.md).</span></span> <span data-ttu-id="9054a-178">Autrement dit, quand vous installez un package dans un projet à partir d’un hôte, NuGet enregistre l’identificateur du package et le numéro de sa version dans cette liste de références.</span><span class="sxs-lookup"><span data-stu-id="9054a-178">That is, whenever you install a package from some host into a project, NuGet records the package identifier and version number in this reference list.</span></span> <span data-ttu-id="9054a-179">(La désinstallation d’un package supprime bien sûr celui-ci de la liste.)</span><span class="sxs-lookup"><span data-stu-id="9054a-179">(Uninstalling a package, of course, removes it from the list.)</span></span> 

![Une liste des références NuGet est créée à l’installation du package et elle peut être utilisée pour restaurer des packages ailleurs.](media/nuget-restore.png)

<span data-ttu-id="9054a-181">Avec seulement la liste des références NuGet peut alors à tout moment réinstaller, &mdash;autrement dit, restaurer&mdash;, tous ces packages à partir d’hôtes publics et/ou privés.</span><span class="sxs-lookup"><span data-stu-id="9054a-181">With only the reference list, NuGet can then reinstall&mdash;that is, restore&mdash;all of those packages from public and/or private hosts at any later time.</span></span> <span data-ttu-id="9054a-182">(Pour cette raison, nuget.org n’autorise pas la suppression définitive des packages publiés, même s’ils peuvent être masqués. Consultez [Suppression de packages](Policies/deleting-packages.md).)
